## Project Overview

This repo is a competition entry for [Prior Labs' World Cup Game Outcome Prediction competition](https://ux.priorlabs.ai/worldcup). The goal is to submit a CSV of match-outcome probabilities (home win / draw / away win) for upcoming fixtures.

**Submission deadline for the Final round:** 19 Jul 2026, 16:00 UTC

## Running Predictions

The virtual environment lives in `tabpfn-football-predictions/.venv/`.

```bash
cd tabpfn-football-predictions
source .venv/bin/activate

# Generate predictions (uses cached results.csv if present)
python predict.py

# Force re-download of the dataset from martj42/international_results
python predict.py --refresh
```

Output is saved to `predictions_YYYYMMDD.csv` in the same directory.

## Submission Format

The competition requires a CSV with exactly these columns (no extras):

```
date,home_team,away_team,p_home_win,p_draw,p_away_win
```

**Important:** `predict.py` outputs an extra `predicted` column — strip it before submitting:

```bash
cut -d',' -f1-3,5-7 predictions_20260719.csv > submission.csv
```

Or in Python:
```python
df[["date","home_team","away_team","p_home_win","p_draw","p_away_win"]].to_csv("submission.csv", index=False)
```

Rules: full country names (e.g. `Argentina`, not `ARG`); each probability in [0,1]; the three probabilities per row must sum to 1.

## Architecture of `predict.py`

Everything runs in one script with four stages:

1. **`load_data()`** — downloads/loads `results.csv` from [martj42/international_results](https://github.com/martj42/international_results). Assigns an `importance` score per tournament (60 for World Cup, 20 for friendlies, etc.) and encodes outcome as `home_win`/`draw`/`away_win`.

2. **`build_features()`** — single chronological forward-pass through all matches. Maintains running state (ELO ratings, recent-form deques, H2H records, last-match date) so every row's features reflect only pre-match information. **No data leakage is possible by construction.** Key globals: `HOME_ADV = 65.0` (ELO home advantage), `TRAIN_START = 2014-01-01`.

3. **`train()`** — fits `TabPFNClassifier` on up to `MAX_TRAIN = 10000` recent played matches. TabPFN is an in-context Bayesian classifier; `ignore_pretraining_limits=True` lifts the default 1000-row cap.

4. **`main()`** — backtests on the prior calendar month (prints accuracy + log-loss), then retrains on the full recent window and predicts all future fixtures (`home_score` is NaN and `date > TODAY`).

### Feature List

| Feature | Description |
|---|---|
| `elo_diff` | ELO gap: home ELO + home advantage − away ELO |
| `home_elo`, `away_elo` | Current ELO ratings (initialized at 1500) |
| `form5_diff` / `form10_diff` | Difference in avg points-per-game over last 5/10 matches |
| `home_winrate`, `away_winrate` | Win rate over last 10 matches |
| `home_gf5`, `away_gf5` | Goals scored per game over last 5 matches |
| `home_ga5`, `away_ga5` | Goals conceded per game over last 5 matches |
| `gd10_diff` | Difference in avg goal difference over last 10 matches |
| `home_streak`, `away_streak` | Current win streak |
| `home_rest`, `away_rest` | Days since last match (capped at 90) |
| `home_played`, `away_played` | Total matches played in history |
| `h2h_n` | Number of head-to-head meetings |
| `h2h_home_winrate` | Home team win rate in H2H |
| `h2h_draw_rate` | Draw rate in H2H |
| `h2h_gd` | Avg goal difference in H2H (from home team's perspective) |
| `neutral` | 1 if played at a neutral venue |
| `importance` | Tournament importance (60=World Cup, 20=friendly) |

## Improving the Model

The baseline achieves ~59% accuracy and ~0.86 log-loss. Areas with room to improve:

- **Additional features:** squad depth, injury/suspension data, altitude, travel distance
- **Alternative models:** XGBoost, LightGBM, or an ensemble alongside TabPFN
- **Calibration:** `sklearn.calibration.CalibratedClassifierCV` to sharpen probability estimates
- **Longer H2H window or decay weighting:** more recent matches weighted higher in form calculations
- **Tournament-aware home advantage:** World Cup is played at neutral venues in 2026 — `neutral=1` should apply to all matches
